# C4 — Multi-Source Incident Fusion & Real-Time Dashboard

**Component 4 of R26-DS-011: AI-Based Emergency Triage System for Sri Lankan Disasters**

600 simulated disaster reports (300 flood + 300 fire) in **Sinhala only**,
fused into deduplicated Unified Incident Records with a real-time operator dashboard.

---

## Quick Start (4 commands)

```bash
pip install -r requirements.txt
python generate_dataset.py
python server.py
# Open http://localhost:8000
```

## Dataset Details

| Item | Value |
|------|-------|
| Total reports | 600 |
| Flood reports | 300 (150 voice + 150 SMS) |
| Fire reports | 300 (150 voice + 150 SMS) |
| Language | Sinhala only (65% Unicode, 35% Romanized) |
| Provinces | Western + Central |
| Districts | Colombo, Gampaha, Kalutara, Kandy, Ratnapura |
| Distinct incidents | ~90 real-world events with duplicates |
| Ground truth labels | Included in `_ground_truth` field |

### Scenario Types

- **Mass-report Critical:** 10-12 duplicate reports about same flood/fire
- **Adjacent Critical:** Two events 0.5-2km apart that must NOT merge
- **Conflicting data:** Same incident with different people counts (3 vs 10)
- **Long time gap:** Same event reported hours apart
- **Noise:** Isolated single reports
- **Cross-variant location:** Same place in Sinhala Unicode + Romanized forms

## Pipeline Architecture

```
C1 (Voice) ──→ JSON ──┐
                       ├──→ Normalize → Gazetteer → Embed → DBSCAN → UIR → Dashboard
C3 (SMS)   ──→ JSON ──┘
```

| Stage | File | Description |
|-------|------|-------------|
| 1-2 | `normalizer.py` | Canonical schema conversion, field aliasing, flag-and-continue |
| 3 | `gazetteer.py` | 31 locations, 177 Sinhala variants, exact→fuzzy→landmark lookup |
| 4 | `embedder.py` | LaBSE (or fallback character n-gram with keyword bridge) |
| 5-6 | `clustering.py` | Urgency-sensitive DBSCAN, 5-condition safety gate, conflict detection |
| 7-8 | `server.py` + `dashboard.html` | FastAPI + WebSocket + React + Leaflet.js |

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Dashboard stuck on "Connecting" | Make sure `python server.py` is running first |
| Port 8000 in use | Change port in server.py last line: `port=8080` |
| `ModuleNotFoundError` | Make sure you're `cd` into the c4_project folder |
| `python` not found | Try `py` or `python3` instead |
| LaBSE download fails | OK — fallback embeddings work automatically |
| Venv won't activate | PowerShell: `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` |
